     1→## 4. Engine (`fitn_engine` package)
     2→
     3→Pure Dart. No Flutter. No I/O. JSON data files loaded from `assets/` at construction.
     4→
     5→**Public API:**
     6→
     7→```dart
     8→// fitn_engine/lib/fitn_engine.dart
     9→
    10→class FitnEngine {
    11→  FitnEngine({required EngineData data});  // data loaded by caller
    12→
    13→  AssessmentResult assessProfile(UserProfile profile);
    14→  FitnessPlan proposePlan(UserProfile profile, AssessmentResult assessment, PlanPreferences prefs);
    15→  GeneratePlanResponse generatePlan(UserProfile profile, PlanPreferences prefs);  // convenience: assess + propose
    16→}
    17→
    18→class EngineData {
    19→  final List<Exercise> exercises;        // 1,217
    20→  final List<SplitDesign> splits;        // 8

    22→  final List<Recipe> recipes;            // ~400 (curated + uncurated + pre/post)
    23→  final Map<String, FoodItem> foodDatabase;
    24→}
    25→
    26→// Loads from rootBundle (Flutter) or File (CLI/test)
    27→Future<EngineData> loadEngineData({String? basePath});

    89→  const factory UserProfile({

    91→    required Sex sex,
    92→    required double heightCm,            // 140–230
    93→    required double weightKg,            // 35–300
    94→    required ActivityLevel activityLevel,
    95→    required TrainingStatus trainingStatus,

   100→    double? bodyFatPct,                  // 2–60, null = unknown
   101→    double? neckCm,                      // 20–80
   102→    double? waistCm,                     // 40–200
   103→    double? hipCm,                       // 40–200 (required for female Navy BF)
   104→    CutRateTier? cutRateTier,
   105→    BulkAggressiveness? bulkAggressiveness,

   149→### 4.2 Assessment sub-engine
   150→
   151→Orchestration (`assessProfile`) runs 4 sub-assessments in order, **never short-circuiting on failure**:
   152→
   153→1. `assessBodyComposition(profile)` — root of the dependency tree.
   154→2. `assessHealthRisk(profile)` — independent of body comp.
   155→3. `assessMuscularPotential(profile, bodyComp.bodyFatPct)` — **skipped** if `bodyComp == null`.
   156→4. `decideStrategy(profile, bodyComp.bodyFatPct)` — **skipped** if `bodyComp == null`, defaults to `maintenance`.
   157→
   158→If any sub-assessment throws, record `"sub_name: ArgumentError: <msg>"` in `errors` and set the sub-result to null. `is_partial = errors.isNotEmpty`. The summary string is built defensively (every nullable field guarded).
   159→
   160→`proposePlan()` throws `PartialAssessmentError` if called with `is_partial == true`.
   161→
   162→#### 4.2.1 Body fat % selection (priority order)
   163→
   164→1. **User-provided** (`bodyFatPct != null`) → use as-is.
   165→2. **US Navy circumference** (Hodgdon & Beckett 1984) — if `hasCircumferenceMeasurements`:
   166→   - Male: `86.010 × log10(waistIn − neckIn) − 70.041 × log10(heightIn) + 36.76`
   167→   - Female: `163.205 × log10(waistIn + hipIn − neckIn) − 97.684 × log10(heightIn) − 78.387`
   168→   - If `(waistIn − neckIn) ≤ 0` (male) or `(waistIn + hipIn − neckIn) ≤ 0` (female), return null → fall through to CUN-BAE.
   169→   - Clamp to `[2.0, 60.0]`.
   170→3. **CUN-BAE** (always computable):
   171→   - `bf = -24.988 + 0.503 × age + 1.0689 × bmi + 0.462 × (sex == female ? 1.0 : 0.0)`
   172→   - Clamp to `[2.0, 60.0]`.
   173→
   174→#### 4.2.2 BMI categories
   175→
   176→| Category     | BMI range            |
   177→| ------------ | -------------------- |
   178→| UNDERWEIGHT  | `[0, 18.5)`          |
   179→| NORMAL       | `[18.5, 25.0)`       |
   180→| OVERWEIGHT   | `[25.0, 30.0)`       |
   181→| OBESE        | `[30.0, +∞)`         |
   182→
   183→#### 4.2.3 Body fat categories (sex-specific)
   184→
   185→| Category     | Male BF%       | Female BF%      |
   186→| ------------ | -------------- | --------------- |
   187→| ESSENTIAL    | `[0, 6)`       | `[0, 14)`       |
   188→| ATHLETE      | `[6, 14)`      | `[14, 21)`      |
   189→| FITNESS      | `[14, 18)`     | `[21, 25)`      |
   190→| ACCEPTABLE   | `[18, 25)`     | `[25, 32)`      |
   191→| OBESITY      | `[25, +∞)`     | `[32, +∞)`      |
   192→
   193→#### 4.2.4 FFMI (Kouri 1995)
   194→
   195→- `ffmKg = weightKg × (1 − bfPct/100)`
   196→- `ffmi = ffmKg / heightM²`
   197→- `normalizedFfmi = ffmi + 6.1 × (1.8 − heightM)`
   198→
   199→**Ceiling constants:** `FFMI_NATURAL_COMMON = 25.0`, `ATTAINABLE = 27.3`, `LIKELY_MAX = 28.0`. Normalization coefficient `6.1`, reference height `1.8 m`.
   200→
   201→**Named target BF% landmarks** (for `targetWeightsKg` map):
   202→
   203→| Label            | Male | Female |
   204→| ---------------- | ---- | ------ |
   205→| athletic         | 14.0 | 21.0   |
   206→| fitness          | 18.0 | 25.0   |
   207→| acceptable       | 25.0 | 32.0   |
   208→| hormonal_floor   | 10.0 | 18.0   |
   209→
   210→`targetWeight = lbm / (1 − targetBf/100)` where `lbm = currentWeight × (1 − currentBf/100)`.
   211→
   212→#### 4.2.5 Health risk
   213→
   214→| Metric | Formula | Classification |
   215→| ------ | ------- | -------------- |
   216→| **WHR** | `waist / hip` | Male: ≤0.85 LOW, ≤0.90 MOD, ≤1.0 HIGH, >1.0 VERY_HIGH. Female: ≤0.80 LOW, ≤0.85 MOD, ≤1.0 HIGH, >1.0 VERY_HIGH. |
   217→| **WHtR** | `waist / height` | Male: <0.50 LOW, <0.53 MOD, <0.58 HIGH, ≥0.58 VERY_HIGH. Female: <0.46 LOW, <0.49 MOD, <0.54 HIGH, ≥0.54 VERY_HIGH. |
   218→| **ABSI** | `(waist_m) × weight^(-2/3) × height^(5/6)` | Z-score from NHANES 10-year age bands (see §9.3). |
   219→| **IBW** | `base + mult × (heightIn − 60)` | 4 formulas: Devine (50/2.3 M, 45.5/2.3 F), Robinson (52/1.9, 49/1.7), Miller (56.2/1.41, 53.1/1.36), Hamwi (48/2.7, 45.4/2.2). |
   220→
   221→**Overall risk** (weighted heuristic): ABSI weight 0.5, WHR 0.3, WHtR 0.2. Add weight when the metric is HIGH/VERY_HIGH (ABSI: ABOVE_AVERAGE/HIGH). Map: ≥0.75 → VERY_HIGH, ≥0.50 → HIGH, any clinical risk factor → MODERATE, else LOW.
   222→
   223→#### 4.2.6 Muscular potential
   224→
   225→- Use **normalized FFMI** for ceiling comparison.
   226→- `ffmiToCeilingPct = clamp(normalizedFfmi / 25.0 × 100, 0, 100)`.
   227→- `headroomKg = max(0, ffmAtCeiling − ffmKg)` where `ffmAtCeiling = (25.0 − 6.1 × (1.8 − heightM)) × heightM²`.
   228→- **Berkhan stage max** (men only): `heightCm − 100` at 5-6% BF. Null for women (essential BF floor is ~10-13%, not 5-6%).
   229→- **Expected monthly muscle gain** (Lyle McDonald, men; × 0.5 for women):
   230→
   231→| Status        | Monthly gain (men) |
   232→| ------------- | ------------------ |
   233→| BEGINNER      | 0.85 kg            |
   234→| NOVICE        | 0.575 kg           |
   235→| INTERMEDIATE  | 0.325 kg           |
   236→| ADVANCED      | 0.10 kg            |
   237→
   238→#### 4.2.7 Strategy decision tree
   239→
   240→**Sex-specific boundaries** (single source of truth):
   241→
   242→| Boundary           | Male | Female |
   243→| ------------------ | ---- | ------ |
   244→| cut_floor          | 10   | 18     |
   245→| bulk_ceiling       | 20   | 28     |
   246→| bulk_start         | 15   | 23     |
   247→| operational_lo     | 10   | 18     |
   248→| operational_hi     | 20   | 28     |
   249→| obese_threshold    | 25   | 32     |
   250→| recomp_excellent   | 23   | 30     |
   251→| recomp_good_lo     | 15   | 25     |
   252→| recomp_limited     | 15   | 25     |
   253→| skinny_fat_lo      | 12   | 20     |
   254→| skinny_fat_hi      | 23   | 31     |
   255→
   256→**Algorithm (first match wins):**
   257→
   258→1. **Safety override (obesity):** `bfPct >= obese_threshold`:
   259→   - BEGINNER + goal != fat_loss → `habitChangeFirst` ("start with habit changes before calorie counting")
   260→   - Else → `cut` ("prioritise fat loss")
   261→2. **Goal = maintenance:**
   262→   - If `bfPct < hormonal_floor` → warn about bulk.
   263→   - If `bfPct > operational_hi` → warn about cut.
   264→   - **Reverse-diet detection:** if `intakeLogKcal.length >= 30` AND `operational_lo <= bfPct <= operational_hi`:
   265→     - Trailing 7-day avg intake vs estimated TDEE (compute via RMR+TDEE pipeline; fallback `weightKg × 35`).
   266→     - If `recentIntake < tdee × 0.90` → `reverseDiet`.
   267→   - Else → `maintenance`.
   268→3. **Goal = strength:** if `bfPct > bulk_ceiling` → `cut`; else `maintenance`.
   269→4. **Goal = fat_loss:** if `bfPct < cut_floor` → `maintenance` (protect hormones); else `cut`.
   270→5. **Goal = muscle_gain:** if `bfPct > bulk_ceiling` → `cut`; else `bulk`.
   271→6. **Goal = recomp:** `bfPct >= recomp_excellent` → `recomp` (excellent); `>= recomp_good_lo` → `recomp` (good); else `bulk` (limited).
   272→7. **Auto-decide (no explicit goal override — fallback path):**
   273→   - `bfPct > operational_hi` → `cut`
   274→   - `bfPct < cut_floor` → `bulk`
   275→   - Novice + skinny-fat range → `recomp`
   276→   - Novice + `bfPct < bulk_start` → `recomp` (milk newbie gains)
   277→   - `bfPct < bulk_start` → `bulk`
   278→   - Else → `cut`

   280→### 4.3 Nutrition sub-engine
   281→
   282→`buildNutritionPlan(profile, assessment, exerciseHours, exerciseIntensity, climate, weightReducedPct)`:
   283→
   284→1. **RMR** — `KATCH_MCARDLE` if `bodyFatPct != null`, else `MIFFLIN_ST_JEOR`.
   285→   - Mifflin: `10 × weight + 6.25 × height − 5 × age + (male ? 5 : -161)`
   286→   - Katch-McArdle: `370 + 21.6 × lbm` (throws if `bfPct ∉ [2, 60]`)
   287→   - Adjustments: `× 0.95` if active deficit (cut/recomp), `× 0.97` if `weightReducedPct > 0.10`.
   288→2. **TDEE** — RippedBody activity factors (1.25 / 1.45 / 1.65 / 1.85 / 2.05 for sedentary / mostly_sedentary / lightly_active / active / highly_active).
   289→3. **Adaptive TDEE** — if `weightLogKg` and `intakeLogKcal` are both non-empty, equal-length, and ≥8 entries:
   290→   - `observed = avgIntake − (Δweight × 7700) / nDays`
   291→   - Sanity: `observed ∈ [800, 7000]`, else skip.
   292→   - Ramp weight: `w = clamp((nDays − 7) / 53, 0, 1)` — no adaptation below 7 days, full at 60+.
   293→   - `adaptive = w × observed + (1 − w) × tdee`.
   294→4. **Calorie targets** — by strategy, with floor 1200 (F) / 1500 (M):
   295→
   296→| Strategy           | Computation                                                                                       |
   297→| ------------------ | ------------------------------------------------------------------------------------------------- |
   298→| cut                | `weeklyLoss = weight × rate`; `dailyDeficit = weeklyLoss × 1100`; `target = tdee − dailyDeficit` |
   299→| bulk               | `monthlyGain = weight × rate`; `dailySurplus = monthlyGain × 330`; `target = tdee + dailySurplus` |
   300→| maintenance        | `target = tdee`                                                                                   |
   301→| recomp             | `target = tdee × (1 − deficitPct)` where deficitPct = 0.15/0.05/0.0 by recomp potential           |
   302→| reverse_diet       | Weekly ladder: `+50/+100/+150 kcal/wk` by aggressiveness, until target reached                    |
   303→| habit_change_first | Same as maintenance                                                                               |
   304→
   305→**Cut rate** (priority): explicit `cutRateTier` → BF%-threshold table → default 0.75% BW/week. Hard cap 1.0% BW/week.
   306→
   307→**Cut rate by BF% threshold** (first match):
   308→
   309→| Sex    | Thresholds (BF%) | Rate (% BW/week)   |
   310→| ------ | ---------------- | ------------------ |
   311→| Male   | [25, 20, 15]     | [1.00, 0.75, 0.50, 0.50] |
   312→| Female | [32, 28, 22]     | [1.00, 0.75, 0.50, 0.50] |
   313→
   314→**Bulk rate** (priority): explicit `bulkAggressiveness × trainingStatus` table → default by status (beginner 2%/mo, novice 1.5%/mo, intermediate 1%/mo, advanced 0.5%/mo).
   315→
   316→**Bulk weekly rate tiers** (% BW/week):
   317→
   318→| Aggressiveness     | Beginner/Novice | Intermediate | Advanced |
   319→| ------------------ | --------------- | ------------ | -------- |
   320→| conservative       | 0.0020          | 0.0015       | 0.0010   |
   321→| happyMedium        | 0.0050          | 0.00325      | 0.0015   |
   322→| aggressive         | 0.0080          | 0.00575      | 0.0035   |
   323→| veryAggressive     | 0.0100          | 0.00800      | 0.0060   |
   324→
   325→5. **Macros:**
   326→   - **Protein:**
   327→     - Obese override (`bfPct >= obese_threshold`): `1 g × heightCm`.
   328→     - BF% known: `lbm_lb × 1.14` (cut) or `× 1.0` (else).
   329→     - BF% unknown: `target_weight_lb × 1.0` (cut, target = 90% current) or `weight_lb × 0.73` (else).
   330→     - Diet boost: vegan ×1.20 (ceil), vegetarian ×1.10 (ceil).
   331→   - **Fat:** midpoint of strategy range (deficit/recomp: 15-25%; surplus/maintenance/reverse: 20-30%). Floor: `max(40, weight_lb × 0.25)`. Saturated ceiling: 10% of calories.
   332→   - **Carbs:** remainder = `targetKcal − proteinKcal − fatKcal`, clamped to ≥0.
   333→6. **Hydration** (multi-step FatCalc):
   334→   - Base: `weightKg × 0.030 L`
   335→   - Sex: +0.3 L (male)
   336→   - Exercise: `hours × sweatRate / 1000` (light 300, moderate 500, intense 800 mL/hr)
   337→   - Climate multiplier (cold 0.95, temperate 1.0, hot 1.3, hot_humid 1.4) applied to exercise component only
   338→   - Pregnancy +0.3 L, breastfeeding +0.7 L (throws if male)
   339→   - Soft ceiling 5.0 L (hyponatremia protection)
   340→7. **Micronutrients:** fiber = `14 g × targetKcal / 1000`. Fruit/veg cups: 2/2 (≤2000 kcal), 3/3 (≤3000), 4/4 (≤4000+).
   341→8. **Timeline:**
   342→   - Cut: `max(4, floor(kgToLose / weeklyRateKg) + 4)` where target BF = `operational_lo + 2`.
   343→   - Bulk: `max(12, floor((targetGainKg / monthlyRateKg) × 4.348) + 4)` where targetGain = 5% BW.
   344→   - Recomp: 12 weeks. Maintenance: 12. Habit-change: 8. Reverse-diet: parse from notes, default 8.
   345→
   346→### 4.4 Training sub-engine
   347→
   348→`buildTrainingPlan(profile, assessment, muscleFocus, programDurationWeeks)`:
   349→
   350→1. **Derive goal** from `(primaryGoal, recommendedStrategy)`:
   351→
   352→| primaryGoal | strategy                                        | → TrainingGoal      |
   353→| ----------- | ----------------------------------------------- | -------------------- |
   354→| strength    | maintenance / bulk / recomp                     | strength             |
   355→| (any)       | cut                                             | fatLoss              |
   356→| (any)       | bulk                                            | muscleGain           |
   357→| (any)       | recomp                                          | recomp               |
   358→| (any)       | maintenance                                     | maintenance          |
   359→| (any)       | habitChangeFirst                                | generalFitness       |
   360→| (any)       | reverseDiet (or unmapped)                       | hypertrophy (fallback) |
   361→
   362→2. **Pick split** from 8 designs (`split_designs.json`):
   363→   - Filter by `days_per_week` → by `experience` AND `goal` → by `experience` → by `goal` → all.
   364→   - Tie-break by experience preference order:
   365→     - BEGINNER: fullBody > upperLower > ppl
   366→     - NOVICE: fullBody > upperLower > ppl > pushPull
   367→     - INTERMEDIATE: upperLower > fullBody > ppl > pplul > pplX2
   368→     - ADVANCED: pplX2 > pplul > bodyPart > upperLower > fullBody
   369→3. **Pick progression:** BLOCK if (strength + intermediate/advanced); else LINEAR (beginner/novice), DUP (intermediate), BLOCK (advanced).
   370→4. **Apply muscle focus:** for each focus muscle, append 1-2 accessory slots to templates that already train that muscle.
   371→
   372→| Focus muscle | Added slots (category, sets, pattern)                                       |
   373→| ------------ | --------------------------------------------------------------------------- |
   374→| chest        | chest_fly accessory × 3; incline_push accessory × 3                         |
   375→| back         | seated_row accessory × 3; vertical_pull accessory × 3                       |
   376→| quads        | knee_extension accessory × 3; front_squat compound_secondary × 3            |
   377→| hamstrings   | knee_flexion accessory × 3; romanian_deadlift accessory × 3                 |
   378→| glutes       | hip_thrust compound_secondary × 3; glute_isolation accessory × 3            |
   379→| shoulders    | lateral_raise accessory × 4; rear_delt accessory × 3                        |
   380→| arms         | elbow_flexion accessory × 4; elbow_extension accessory × 4                  |
   381→| biceps       | elbow_flexion accessory × 4                                                 |
   382→| triceps      | elbow_extension accessory × 4                                               |
   383→| calves       | ankle_plantarflexion accessory × 5                                          |
   384→| abs          | core_anti_extension accessory × 3; core_anti_rotation accessory × 3         |
   385→
   386→5. **Build workouts from templates** — for each slot, run the **7-tier exercise selector**:
   387→   - Equipment filter (full_gym: 33 types, home_gym: 9, bodyweight_only: 2).
   388→   - Exclude CARDIO/MOBILITY categories; exclude plyometrics for beginners.
   389→   - Tier 1-7: progressively relax constraints (pattern → muscle → category), experience enforced or relaxed.
   390→   - Sort by: equipment preference rank → experience distance → view count → name.
   391→   - Movement pattern detection: keyword match (slug wins 100+length, name wins length) → force-type fallback → primary-muscle fallback.
   392→6. **Build mesocycles:**
   393→   - Mesocycle length: 4w (beginner/novice), 5w (intermediate), 6w (advanced).
   394→   - Program duration: 4/8/10/12w by experience (bumped to 12 if strength + intermediate/advanced). User override: 4-52w.
   395→   - `numMesocycles = max(1, floor(duration / mesoLength))`.
   396→   - Each mesocycle's last week is a deload (sets × 0.5, RPE − 1.5).
   397→   - BLOCK progression: assign phases across mesocycles — 1→[accumulation], 2→[accumulation, intensification], 3+→[accumulation, intensification, peak].
   398→7. **Apply periodization** (5 layers, in order):
   399→   - **Layer 1 — Goal preset** (base reps/rest/RPE by goal × exercise category).
   400→   - **Layer 2 — DUP day-type modifiers** (heavy/moderate/light, only if `dayType` set on template).
   401→   - **Layer 3 — Block phase modifiers** (accumulation/intensification/peak; for strength + compound_primary, use StrengthPhase specs).
   402→   - **Layer 4 — Deload** (sets × 0.5, RPE − 1.5).
   403→   - **Layer 5 — RIR clamp** (lookup RIR range by tier × rep range, convert to RPE bounds, clamp).
   404→8. **Compute weekly volume** (fractional set counting: primary 1.0, secondary 0.5, dedupe secondary against primary).
   405→9. **Validate volume** against landmarks (MEV/MAV/MRV/ML for 22 muscles + default fallback).
   406→10. **Per-session cap:** 11 sets/muscle/workout — warn if exceeded.

   459→### 4.5 Meal plan sub-engine
   460→
   461→`buildMealPlan(profile, assessment, nutrition, mealFrequency, cuisinePreference, allergensToAvoid, excludedIngredients, includePrePostWorkout)`:
   462→
   463→1. **Compute slot targets** from daily macros × meal allocation:
   464→
   465→| Frequency | Allocation                                              |
   466→| --------- | ------------------------------------------------------- |
   467→| 2 meals   | LUNCH 0.50, DINNER 0.50 (no breakfast — intentional)   |
   468→| 3 meals   | BREAKFAST 0.30, LUNCH 0.35, DINNER 0.35                 |
   469→| 4 meals   | BREAKFAST 0.25, LUNCH 0.30, DINNER 0.30, SNACK 0.15    |
   470→| 5 meals   | BREAKFAST 0.20, LUNCH 0.25, DINNER 0.30, SNACK 0.15, SNACK 0.10 |
   471→
   472→2. **Determine training days** for the 7-day week:
   473→   - 2d → [1, 4], 3d → [1, 3, 5], 4d → [1, 2, 4, 5], 5d → [1, 2, 3, 4, 5], 6d → [1, 2, 3, 4, 5, 6].
   474→3. **For each of 7 days × N slots:** allocate meal via `scoreRecipeForSlot` → pick top → scale → fill gap.
   475→4. **Track recipe usage** (rolling 3-day and 7-day windows) for variety scoring.
   476→
   477→**Diet tag derivation:**
   478→- omnivore → `"OMNI"`
   479→- vegan → `"VEGAN"`
   480→- vegetarian → `"VEGAN"` (recipe DB has no separate vegetarian tag — vegetarians get vegan recipes)
   481→
   482→**Recipe scoring** (9 weighted components, each 0-100):
   483→
   484→| Component        | Weight | Tight band | Loose band |
   485→| ---------------- | ------ | ---------- | ---------- |
   486→| kcal_match       | 26     | ±20%       | ±40%       |
   487→| protein_match    | 22     | ±15%       | ±30%       |
   488→| carb_match       | 13     | ±20%       | ±40%       |
   489→| fat_match        | 9      | ±25%       | ±50%       |
   490→| diet_match       | 13     | —          | —          |
   491→| goal_fit         | 4      | —          | —          |
   492→| fiber_match      | 4      | ±50%       | ±100%      |
   493→| variety_bonus    | 4      | —          | —          |
   494→| cuisine_match    | 5      | —          | —          |
   495→
   496→**Hard exclusions** (score = 0):
   497→- Allergen violation
   498→- Excluded ingredient present
   499→- Diet mismatch
   500→
   501→**Allergen scanning** (best-in-class — plant-qualifier suppression):
   502→- Normalize allergen aliases (`tree_nuts` → `nuts`, `crustacean(s)` → `shellfish`).
   503→- Build word-boundary regexes for each allergen's keywords.
   504→- Sanitize ingredients: blank out plant-named phrases (eggplant, butter lettuce, coconut milk, etc. — full list in §9.2) before scanning for `dairy` and `eggs`.
   505→- For each `dairy`/`eggs` match, check 25-char context before for plant qualifiers (almond, soy, oat, coconut, etc. — full list in §9.2). If found, skip the match.
   506→- Other allergens scan the unsanitized string.
   507→
   508→**Recipe scaling:**
   509→- `scaleFactor = clamp(targetKcal / recipeKcal, 0.7, 1.5)` (1.0 if within ±10%).
   510→- Scaled nutrition = per-serving × scaleFactor.
   511→
   512→**Filler system** (closes macro gap after scaling):
   513→
   514→| Priority | Type     | Trigger (`gap >`) | OMNI options                                           | VEGAN* options                                |
   515→| -------- | -------- | ----------------- | ------------------------------------------------------ | --------------------------------------------- |
   516→| 1        | Protein  | 5 g               | Whey, Greek yogurt, egg white, cottage cheese, chicken | Tofu, tempeh, pea/soy protein, lentils        |
   517→| 2        | Carb     | 5 g               | White/brown rice, oats, banana, whole wheat bread, quinoa, sweet potato | (same)                                        |
   518→| 3        | Fat      | 3 g               | Olive oil, almonds, peanut butter, walnuts, avocado    | (same)                                        |
   519→| 4        | Veg      | 3 g (main meals only) | Broccoli, spinach, mixed greens, bell pepper, asparagus, green beans, carrots | (same)                                        |
   520→
   521→\* VEGAN prefix match — `VEGAN_ETHIOPIAN` resolves to VEGAN list.
   522→
   523→Filler grams: `(targetG / food.per100g) × 100`, capped at `serving × {protein:4, carb:3, fat:3}`, min serving = `serving × 0.5`. Veg: fixed `[80g, 200g]`.
   524→
   525→**Allergen → filler exclusion map:**
   526→- dairy → Whey, Greek Yogurt, Cottage Cheese, Milk, Cheddar
   527→- eggs → Egg White, Eggs
   528→- gluten → Whole Wheat Bread, Oats
   529→- nuts → Almonds, Walnuts
   530→- peanuts → Peanut Butter
   531→- soy → Tofu, Tempeh, Soy Protein, Edamame
   532→
   533→**Swap system:**
   534→- **Recipe swaps:** alternatives sharing ≥1 meal_type, diet-compatible, kcal within ±20%, no allergens, cuisine-matching if specified. Sorted by kcal closeness, limited to 5.
   535→- **Ingredient swaps:** static substitution database (proteins, carbs, Ethiopian ingredients, fats, dairy alternatives, vegetables). Longest-key-first matching, plant-named-phrase protection.
   536→
   537→**Recipe database** (~400 recipes from 3 JSON files merged at load):
   538→- `recipe_database.json` (79 curated) — `is_curated=true`, notes get `[curated]` tag.
   539→- `recipe_database_uncurated.json` (~305) — `is_curated=false`. ID collisions with curated → rename to `U<id>`.
   540→- `pre_post_workout_recipes.json` (16) — 4 diets × 2 meal types × 2 kcal bins. `nutrition_source="calculated"`, `source="engine-generated"`.
   541→
   542→**Load-time sanitization:**
   543→- VEGAN-tagged recipe with meat/dairy/egg ingredients → append `[diet-warning: tagged VEGAN but ingredients contain meat/dairy/egg — likely a curation error]`.
   544→- Stated kcal vs macro-derived kcal (P×4 + C×4 + F×9) > 10% off → append `[kcal-warning: ...]`.
   545→
   546→**Meal name templates** (cycled by day for variety):
   547→- 5-meal canonical order: BREAKFAST, SNACK, LUNCH, SNACK, DINNER (interleaved — NOT snacks at end).
   548→- Each meal type has 7 default name options (e.g. breakfast: Greek Yogurt Parfait, Oatmeal Power Bowl, Egg & Avocado Toast, ...).
