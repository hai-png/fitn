## Structure
```
fitn/
  fitn_engine/          # Pure-Dart engine package (the "first deliverable")
    lib/
      fitn_engine.dart  # public API
      src/
        models/         # freezed-style models (we'll use plain Dart classes since codegen not available)
        assessment/     # body comp, health risk, muscular potential, strategy
        nutrition/      # RMR, TDEE, calories, macros, hydration, micros
        training/       # split design, exercise selector, periodization, volume landmarks
        meal_plan/      # allocator, recipe loader, scorer, scaler, swap system
        utils/          # rounding, units, enum helpers
        errors.dart
        version.dart
    assets/             # JSON data files (already has all_exercises.json + recipe_database_uncurated.json)
    test/
  fitn_app/             # Flutter app
    lib/
      main.dart, app.dart, router.dart, bootstrap.dart
      core/             # env, result, extensions
      engine/           # engine provider + plan generation
      data/             # Isar collections + repos, Supabase sync, prefs
      state/            # Riverpod notifiers (app, plan, workout session, progress, auth, sync)
      ui/               # theme, widgets, shell, onboarding, tabs, workout, library, auth, settings
    test/
  supabase/             # SQL schema
  README.md
```

## Naming conventions
- Dart `camelCase` for fields, `snake_case` on the wire (`@JsonKey`).
- Pure-Dart models in the engine: plain Dart classes with `fromJson` / `toJson` / `copyWith` (no freezed codegen since this is a fresh project without `dart pub run build_runner`).
- Engine version: `3.2.0`.
- Banker's rounding everywhere except rep-range math (round-half-up).
